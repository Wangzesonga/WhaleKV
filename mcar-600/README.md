# MCAR-600: Multi-turn Compression Attention Rot Benchmark

> **Part of the WhaleKV paper:** "WhaleKV: Navigating the Deep Context Ocean via Topological KV Cache Compression"

MCAR-600 is a diagnostic benchmark that measures KV cache compression quality under **adversarial multi-turn attention drift**. Unlike standard long-context benchmarks, MCAR-600 is specifically designed to expose *Compression-induced Attention Rot*: the progressive loss of early critical context as repeated evictions shift attention toward recent, irrelevant dialogue.

---

## Dataset Summary

| Attribute | Value |
|-----------|-------|
| **Total samples** | 600 (NARR 200 + MEET 200 + CODE 200) |
| **Turns per sample** | 6 (context + shallow + 3 distractors + dual-probe) |
| **Context length** | 15K–25K tokens (mean ≈ 18.5K) |
| **Domains** | Narratives (fiction), Meetings (transcripts), Code (technical docs) |
| **Distractor reference_assistants** | **Claude Sonnet 4.6** (all 600 samples × 3 turns = 1,800 entries) |
| **T1 reference_assistants** | meta-llama/Llama-3.1-8B-Instruct (preserved from original) |
| **probe_b.reference_answer** | meta-llama/Llama-3.1-8B-Instruct (preserved from original) |
| **Language** | English only |

---

## What Makes MCAR-600 Different

```
Turn 0 ── Long document (15-25K tokens, with two injected "islands")
Turn 1 ── Shallow query about the document         ← activates attention on document
Turn 2 ── Unrelated distractor (cross-domain)      ← first attention drift
Turn 3 ── Unrelated distractor (cross-domain)      ← second attention drift
Turn 4 ── Unrelated distractor (cross-domain)      ← third attention drift
Turn 5 ── Dual-probe: (A) exact token recall + (B) causal comprehension
```

The three cross-domain distractor turns simulate realistic multi-turn conversations where users switch topics after a long document read. Each cache compression step shifts attention away from early document content, and MCAR-600 measures what survives.

### Reference Assistants

Turns 2–4 are **distractor conversations** completely unrelated to the document. They require a realistic assistant response to grow the KV cache naturally. This distribution includes 100 unique question templates:

- Business/strategy analysis (market entry, pricing, cloud providers, etc.)
- Technical explainers (APIs, Kubernetes, neural networks, etc.)
- Creative writing (stories, scripts, songs, etc.)
- Lifestyle advice (cooking, fitness, travel, etc.)

**All 1,800 distractor reference_assistants** (600 samples × 3 distractor turns) in this release were **generated by Claude Sonnet 4.6** on 2026-05-25 and are marked in `inference_spec.claude_distractor_generation`.

## Evaluation Metrics

| Metric | Type | Description |
|--------|------|-------------|
| **IS** (Island Survival) | `exact_substring` (0/1) | Whether the injected exact token survives compression and is reproduced in the final answer |
| **HR** (Hub Retention) | `llm_judge_1-5 / 5` | Whether the model retains causal/structural comprehension of the document (LLM judge, normalized to [0,1]) |
| **PS** (Pareto Score) | `mean(IS_i × HR_i)` | **Primary metric** — per-sample product averaged; neither dimension can be sacrificed |
| **Δ** (Retention Bias) | `mean(HR) − mean(IS)` | >0: globally biased; <0: precision-biased; ~0: balanced |
| **Degradation** | `PS(r=0.8) − PS(r=0.2)` | Multi-run metric (Table 10): how much PS drops under extreme vs mild compression |

> **PS formula note:** PS is computed as the mean of per-sample products `IS_i × HR_i`, which correctly weights each sample's combined performance. This differs from `mean(IS) × mean(HR)` (product of averages) — both values are close but the per-sample mean is the paper-correct definition.

## Quick Start

### 1. Install dependencies

```bash
pip install transformers accelerate openai tqdm
```

### 2. Load the dataset

```python
import json

samples = []
for domain in ['narr', 'meet', 'code']:
    with open(f'data/mcar_600_{domain}.jsonl') as f:
        samples.extend([json.loads(l) for l in f])

print(f"Loaded {len(samples)} samples")
# Loaded 600 samples
```

Or load all at once:

```python
with open('data/mcar_600_merged.jsonl') as f:
    samples = [json.loads(l) for l in f]
```

### 3. Understand sample structure

```python
s = samples[0]

# Metadata
print(s['sample_id'])            # e.g. "MCAR-NARR-000"
print(s['domain'])               # "narr" | "meet" | "code"
print(s['island_metadata'])      # Positions and types of injected evidence

# Turns
for turn in s['turns']:
    print(turn['turn_idx'], turn['turn_type'], turn['content'][:60])
# 0  context_injection     *** START OF PROJECT GUTENBERG ...
# 1  shallow_activation    What weather or natural environment is desc...
# 2  attention_distractor_1  Draft dialogue between two strangers meeting...
# 3  attention_distractor_2  What are quick but nutritious breakfast optio...
# 4  attention_distractor_3  What is technical debt and how do teams manage...
# 5  dual_probe            Please answer two questions about the first text:

# Ground truths
probe_turn = s['turns'][5]
```

### 4. Run evaluation (MCAR-Standard Protocol)

```python
from src.run_mcar_standard import run_mcar_standard, save_mcar_results
from transformers import AutoModelForCausalLM, AutoTokenizer
import whalekv

# Load your model
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Meta-Llama-3.1-8B-Instruct",
    device_map="auto",
    torch_dtype="bfloat16"
)
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Meta-Llama-3.1-8B-Instruct")

# Choose your compression method
press = whalekv.WhaleKVAdaptivePress(compression_ratio=0.5)
# press = whalekv.SnapKVPress(compression_ratio=0.5)

# IMPORTANT: Apply KVPress as a context manager BEFORE run_mcar_standard.
# The press hooks into model.generate() automatically inside the context.
# For full-cache baseline (no compression), omit the 'with' block.
with press(model):
    results = run_mcar_standard(
        model=model,
        tokenizer=tokenizer,
        samples=samples,
        use_reference_assistant=True,  # Use fixed Claude/Llama transcripts (recommended)
        device="cuda",
    )

# Save results (predictions always include sample_id for matching)
save_mcar_results(results, output_dir='./results', method_name='WhaleKVAdaptive_r0.5')
```

## MCAR-Standard Protocol (Required for Valid Results)

```
                 ┌─────────────────────────────────────────────┐
                 │  MCAR-Standard: Turn-by-Turn KV Reuse       │
                 └─────────────────────────────────────────────┘

kv = prefill(turn0_text)                         # C0: long document

for t in [1, 2, 3, 4]:
    kv.append(user_turn[t])
    assistant = generate(kv,                     # MUST generate (not skip)
                         max_new_tokens=128,
                         temperature=0.0)
    kv.append(assistant)                         # MUST append assistant KV
    kv = compress(kv)                            # Apply compression here

kv.append(turn5_dual_probe)
answer = generate(kv, max_new_tokens=256, temperature=0.1)
score(answer)
```

**Hard constraints:**

| Constraint | Value |
|------------|-------|
| T1–T4 generation | `max_new_tokens=128, temperature=0.0, top_p=1.0` |
| T5 generation | `max_new_tokens=256, temperature=0.1, top_p=1.0` |
| KV reuse | Must reuse C0 across all turns |
| Assistant T1–T4 | Must be generated (or use fixed reference transcripts) |
| Compression timing | After each assistant generation |

> **Using reference assistants (recommended):** Set `use_reference_assistant=True` in `run_mcar_standard`. The fixed Claude Sonnet 4.6 distractor transcripts and Llama T1 transcripts are injected directly into the KV cache, ensuring cross-method comparability.

## Reference Assistant Generation Details

The `inference_spec.claude_distractor_generation` field in each sample records:

```json
{
  "generator_model": "claude-sonnet-4-5",
  "generator_label": "Claude Sonnet 4.6",
  "generation_date": "2026-05-25",
  "gen_config": {"max_new_tokens": 128, "temperature": 0.0, "top_p": 1.0},
  "note": "Distractor turns (T2-T4) reference_assistants regenerated by Claude Sonnet 4.6. T1 (shallow_activation) and probe_b.reference_answer retain original meta-llama/Llama-3.1-8B-Instruct annotations."
}
```

To regenerate distractor assistants using a different model, use:

```bash
python scripts/generate_distractor_assistants.py \
    --model meta-llama/Llama-3.1-8B-Instruct \
    --input  data/mcar_600_merged.jsonl \
    --output data/mcar_600_merged_llama.jsonl
```

## Integration with WhaleKV

MCAR-600 was designed to evaluate the WhaleKV compression methods:

```python
import whalekv

# Single-turn (no GTA)
press = whalekv.WhaleKVPress(compression_ratio=0.5)

# Multi-turn with fixed β
press = whalekv.WhaleKVMultiTurnPress(compression_ratio=0.5, beta=0.3)

# Multi-turn with entropy-adaptive Φ (recommended for MCAR)
press = whalekv.WhaleKVAdaptivePress(compression_ratio=0.5)

# Baselines
press = whalekv.SnapKVPress(compression_ratio=0.5)
press = whalekv.ChunkKVPress(compression_ratio=0.5)
press = whalekv.PyramidKVPress(compression_ratio=0.5)
```

WhaleKVAdaptivePress is particularly well-suited for MCAR because its entropy-based Φ naturally elevates GTA weight during cross-domain distractor turns (high attention entropy) and reduces it during precise retrieval (low entropy).
